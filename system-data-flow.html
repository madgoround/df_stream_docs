Data Flow
두 개의 Single Source of Truth
인프라 설정과 데이터 정의, 두 가지 SSoT 원칙으로 시스템을 통제합니다.
원칙 1 — 인프라 SSoT: 모든 파이프라인 설정의 원본은 DF-Stream DB에 있다. 외부 서비스(Benthos, Kafka, SR)는 DB의 내용을 반영한 결과물일 뿐이다.
원칙 2 — 의미론 SSoT: 모든 데이터의 정의와 명칭의 원본은 표준 사전(data_standard)에 있다. 각 엔티티와 코드 테이블은 표준 사전의 정의를 참조할 뿐이다.
❌ 금지: Benthos/Kafka/SR 직접 조작 금지 | 새 코드 테이블(_ct) DDL 생성 금지 → 표준 사전 카테고리 추가 | 태그에 자유 텍스트로 속성 분류 금지

SSoT Exception
4-1. DB SSoT 예외 — KEPServerEX 노드 관리
KEPServer 노드(Channel/Device/Tag)는 DB SSoT 정책의 명시적 예외입니다. KEPServer가 원본이고, DB는 캐시(미러)입니다.
역방향 원칙: KEPServer 노드는 KEP Config API를 먼저 거친 후 DB에 반영합니다. KEP 실패 시 DB 변경 없음으로 정합성을 보장합니다.

1. 태그 (Tag) 설비 측정값의 최소 단위 데이터 포인트

설비에서 발생하는 측정값을 식별하는 최소 단위 데이터 포인트. OPC UA 노드 트리의 Leaf(말단) 노드이며, PLC/센서의 개별 레지스터와 1:1로 매핑된다. 컨텍스트(어떤 설비의, 어떤 위치의) + 규칙(어떤 데이터 타입, 어떤 범위)이 부여되어야 의미 있는 데이터가 된다.

식별이름, 전체 경로, 노드 IDTT_Zone1_PV, Channel1.Device1.TT_Zone1_PV

연결PLC 메모리 주소, 데이터 타입, 스캔 주기%MW100, Float, 1000ms
의미표준 용어, 도메인(단위/범위), 속성 분류"온도", ℃, 0~500
